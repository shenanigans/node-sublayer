Incoming Action requests, whether from http or Socket.io are mapped to a Request instance. If bodySchema and/or querySchema are set, the body and/or query properties can be assumed to be conformant by the time the reaction Function is called.

Member Properties

| Object | undefined body

The request body. Form requests are mapped to a simple flat Object and validated by the body schema if specified. JSON bodies are parsed and validated. If the binary streams option is set, the body may be a readable stream. These actions should type-check with instanceof and dispatch accordingly, or use the never buffer option.
| String | undefined contentType

If the remote client has provided it, the value of the Content-Type header is included whenever stream is used.
| Array | undefined files

When cacheFiles is set, an Array of file info Objects containing Buffers is passed. This is part of an unstable feature.
| String method

The method used to access this Action. Used when the Action is mounted without a method specified.
| Array params

If the route regex selected text after the route pattern or the route regex contains matching groups, the additional matched strings are passed in an Array.
| Object query

Request url terms, GET parameters, etc. mapped as a simple flat Object and validated by the query schema if specified.
| stream.Readable | undefined stream

When binaryStreams is set, content types other than lightweight forms and JSON will be streamed instead of cached. If you do not consume the body before replying the incoming body stream is automatically closed with the 'Connection:close' header.
